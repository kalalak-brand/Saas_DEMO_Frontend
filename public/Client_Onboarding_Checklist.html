📋 Client Onboarding Document

Client Setup Checklist &
Information Requirements

Everything you need to collect from the hotel client before deploying the Department Efficiency Analyzer.

01
🏨 Hotel Identity & Details
Basic information to register the hotel in the system
  • ☐
    Hotel Name REQUIRED
    Full official name of the hotel as it should appear in the system and on guest feedback forms.
    Example
    TAJ HOTEL Mumbai • Grand Hyatt New Delhi • Lemon Tree Premier, Goa
  • ☐
    Hotel Code REQUIRED
    A short unique identifier code (3-10 characters). Used in URLs, QR codes, and internal references.
    Example
    TAJH81 • GHND01 • LTPGOA
  • ☐
    Address REQUIRED
    Full street address of the property.
  • ☐
    City & Country REQUIRED
    City and country for location grouping and reporting.
  • ☐
    Contact Email REQUIRED
    Primary contact email for the hotel management. Used for system notifications.
  • ☐
    Contact Phone OPTIONAL
    Hotel management phone number.
  • ☐
    Hotel Logo RECOMMENDED
    High-resolution logo file (PNG/SVG, transparent background, min 200×200px). Displayed on the guest feedback form and dashboard.
02
🏗️ Departments (Categories) to Track
Which departments will have separate feedback forms and analytics?
  • ☐
    List of Departments REQUIRED
    Ask the client: "Which departments in your hotel do you want to separately measure guest satisfaction for?" Each department gets its own QR code, feedback form, and analytics section.
    Common hotel departments
    Department Covers QR Placement
    🛏️ Room / Housekeeping Room cleanliness, amenities, maintenance Bedside table, room door
    🛎️ Front Office / Reception Check-in/out, concierge, billing Reception desk, lobby
    🍽️ Food & Beverage Restaurant food quality, service, hygiene Restaurant table tent
    🏊 Pool & Recreation Pool cleanliness, facilities, safety Pool entrance area
    💆 Spa & Wellness Spa service quality, ambiance Spa reception
    🅿️ Valet & Parking Parking service, valet speed Parking area
    🎪 Events & Banquets Event management, setup quality Banquet hall entrance
  • ☐
    Department Display Order OPTIONAL
    In what order should departments appear in the sidebar/navigation? Default: alphabetical.
  • ☐
    Guest Info Fields per Department REQUIRED
    For each department, which guest details should the form ask for? Check which apply:
    Field Description Common For
    👤 Guest Name Name of the person submitting feedback All departments
    📱 Phone Number Contact number for follow-up All departments
    🔢 Room Number Which room the guest is staying in Room, Front Office
    ✉️ Email Guest email for follow-up Front Office, Events
03
❓ Feedback Questions per Department
The most critical part — these determine what you'll measure
  • ☐
    Questions List per Department REQUIRED
    Ask the client: "For each department, what specific questions do you want to ask your guests?" Ideally 5-10 questions per department. Each question has a type:
    • Rating Question — Guest rates on 1-5 stars (e.g., "Was the room clean on arrival?")
    • Yes/No Question — Guest answers yes or no (e.g., "Did you face any issues during your stay?")
    Example — Room Department Questions
    1. Was the room clean on arrival? (Rating ⭐)
    2. Was daily housekeeping timely? (Rating ⭐)
    3. Did all room facilities work properly? (Rating ⭐)
    4. Was the staff courteous? (Rating ⭐)
    5. Was noise or disturbance minimal? (Rating ⭐)
    6. Did you face any issues during your stay? (Yes/No)
  • ☐
    Composite Scores (KPIs) RECOMMENDED
    Ask: "Do you want to group certain questions into a combined score (KPI)?" For example, grouping "room cleanliness" + "bathroom cleanliness" + "housekeeping timeliness" into a single "Housekeeping Quality Score".
    Example composites
    Room Cleanliness Score = Q1 + Q2 + Q5
    Staff Service Index = Q4 + Q6
    Overall Room Quality = All rating questions averaged
04
👥 User Accounts
Who needs access to the dashboard?
  • ☐
    Admin User REQUIRED
    At least one admin account is needed. This person will manage questions, view analytics, generate QR codes, and configure settings.
    Collect from client
    Field Example
    Full Name Rajesh Kumar
    Username admin@hotelmumbai
    Password (client's choice or auto-generated)
  • ☐
    Viewer / Manager Accounts OPTIONAL
    Ask: "Are there department heads or managers who should have read-only dashboard access?" These users can view analytics but can't modify questions or settings.
    Example viewer accounts
    • F&B Manager — fb_manager / password
    • Front Office Manager — fo_manager / password
    • General Manager — gm_viewer / password
  • ☐
    How Many Total Users? REQUIRED
    Confirm total number of user accounts needed. Typically: 1 Admin + 2-5 Viewers = 3-6 accounts per hotel.

06
📱 QR Code Placement Plan
Where will QR codes be placed in the hotel?
  • ☐
    QR Locations per Department REQUIRED
    For each department, where will the QR code be physically placed? This determines how many QR printouts are needed.
    Typical QR placements
    Department Placement Qty Needed
    Room Bedside table card / bathroom mirror sticker 1 per room (e.g., 200)
    Front Office Reception desk standee 2-3 at lobby
    F&B Table tent / menu card footer 1 per table (e.g., 50)
    Spa Treatment room card 1 per room
  • ☐
    QR Design Preferences OPTIONAL
    Does the client want branded QR codes (logo embedded) or plain? Do they need a specific card/standee size?
07
🚀 Deployment & Infrastructure
Technical requirements — handle internally, but confirm with client
  • ☐
    Domain / URL REQUIRED
    Will the app be hosted on a custom domain? Or a subdomain under your platform?
    Options
    Option A (Subdomain): tajmumbai.acculead.com
    Option B (Custom): feedback.tajhotels.com (client provides DNS access)
    Option C (Shared): app.acculead.com (multi-tenant, hotel selected at login)
  • ☐
    Hosting Region RECOMMENDED
    Where are most guests/staff accessing from? Determines server region for best performance (e.g., Mumbai, Singapore, US East).
  • ☐
    SSL Certificate REQUIRED
    HTTPS is mandatory. If custom domain → need SSL setup. If subdomain → handled by you.
  • ☐
    Database (MongoDB) REQUIRED
    Setup decision: shared MongoDB Atlas cluster (multi-tenant) or dedicated cluster per client.
    Recommendation
    Use MongoDB Atlas M10+ on the nearest region. Each hotel's data is isolated by hotelId field.
  • ☐
    Environment Variables REQUIRED
    Configure these before deployment:
    Required .env variables
    dataBaserURL — MongoDB connection string
    JWT_SECRET — Random secret key for auth tokens
    JWT_EXPIRES_IN — Token expiry (e.g., "90d")
    PORT — Server port (default: 5000)
    NODE_ENV — "production"
08
📦 Handover Deliverables
What you give to the client after setup
  • ☐
    Admin Login Credentials REQUIRED
    Provide the admin and viewer usernames/passwords securely. Recommend the client change passwords on first login.
  • ☐
    Dashboard URL REQUIRED
    The URL where admins log in and view analytics.
  • ☐
    QR Code Files (Print-Ready) REQUIRED
    Generated QR codes for each department in high-resolution PNG/SVG. Include the public review URL for each department.
    Example URLs
    https://app.acculead.com/review/room
    https://app.acculead.com/review/front-office
    https://app.acculead.com/review/food-and-beverage
  • ☐
    Quick Start Guide RECOMMENDED
    A simple 1-page PDF or HTML doc explaining: how to log in, where to view scores, how to add/edit questions, and how to read the dashboard charts.
  • ☐
    Training Session RECOMMENDED
    Schedule a 30-minute video call with the hotel admin to walk through the dashboard, question management, and QR code placement.
  • ☐
    Support Contact REQUIRED
    Provide the client with a support email/phone for post-deployment issues.
⚡
Your Internal Setup Steps (After Collecting Info)
The developer workflow to configure and deploy for a new client
  1. 1
    Create the Hotel — Super Admin dashboard → Hotels → Add Hotel (name, code, address, contact)
  2. 2
    Create User Accounts — Super Admin → Users → Create admin + viewer accounts, assign to the hotel
  3. 3
    Create Departments (Categories) — Log in as hotel admin → Management → create each department with name, slug, icon, guest info fields
  4. 4
    Add Questions — For each department, add the client's questions (rating or yes/no type)
  5. 5
    Create Composites — Group questions into composite scores (KPIs) for the dashboard
  6. 6
    Configure Theme — Settings → set brand color, welcome/thank you messages
  7. 7
    Generate & Print QR Codes — Generate QR codes for each department → export as print-ready files
  8. 8
    Test Everything — Submit test reviews via each QR → verify they appear on the dashboard → check scores
  9. 9
    Handover — Share credentials, QR files, quick start guide → schedule training call → go live! 🎉
